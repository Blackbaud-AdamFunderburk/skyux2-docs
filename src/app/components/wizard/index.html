A wizard guides users through a set of pre-defined steps in a particular order. It uses a progress indicator component within a modal component to represent the steps in a task as a series of related decisions in a userâ€™s direct workflow. The wizard illustrates the step-wise nature of a task and reinforces the progress towards the final goal as users complete each step. Wizards are most useful when users must perform steps in a particular order.

To create a wizard, place a progress indicator inside a modal and set its displayMode property to Horizontal. For more information about progress indicators, see the progress indicator component, And for more information about modals, see the modal component.

Specifies whether to display the progress indicator vertically or horizontally. For modal wizards, you must set the display mode to Horizontal. For information about other types of progress indicators, see the progress indicator component.

The progress indicator nav button component allows users to navigate the steps in modal wizards. We recommend against using it in waterfall pages and passive progress indicators.

Specifies the label to display on the nav button. Specifies the type of nav button to include. Options include finish, next, and previous. Indicates whether to disable the nav button. Specifies the progress indicator component to associate with the nav button.

A wizard is appropriate for forms that meet the following criteria:

  • The task can be divided into discrete steps.
  • The order of the steps is important.
  • User actions in earlier steps affect later steps.
  • Users must complete or review the fields in every step.
  • Users might want to save their progress so that they can leave the form and return later.
