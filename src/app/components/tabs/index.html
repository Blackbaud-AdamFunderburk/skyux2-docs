The tabs module contains components to render tabs that divide large subsets of content on a page. The module includes options to display buttons in the tab area and to make tabs closeable. The sky-tabset component creates a container for the tabs, and within it, the sky-tab component creates an individual tab. The content within a sky-tab component is displayed when that tab is active.

Use when

  • Users need quick access to different slices of the same data.
  • Users need to complete different tasks on the page, which each have a significant amount of data related to the task.
Do use tabs to show different slices of the same data.
Do use tabs to group content for different tasks.

Don't use when

  • You already have a set of tabs on the page. Try changing the workflow rather than using multiple sets of tabs.
Don't use multiple sets of tabs on a single page.
Don't combine vertical tabs with horizontal tabs.

Count

When you use tabs to hold lists, you can include the list's record count in the tab so users know if there are records to look at without navigating to the tab.

Open and close

If users need to work with a subset of large number of available tabs, let them open and close tabs so they are only working with relevant tabs. If users have more than one option of what to display in a new tab, use action buttons within the tab view to let them choose the option they want.

Overflowing tabs

If there are too many tabs to fit in the allotted space, the tabs collapse into a dropdown.

Page titles

If tabs are the main navigation on a page, display the tab header in the page title. Use the following format: <Tab title> - <Page title> - <Product title>. For example: Portfolio - Work center - Blackbaud. In addition, follow these guidelines for page titles:

  • Keep titles short.
  • Make titles unique.
  • Provide concise descriptions of each page's contents or purpose. Titles should stand on their own and make sense out of context.
  • Start with the most meaningful, differentiating terms so that they can be scanned in browser tabs, window lists, and browser bookmarks.
  • Use $sky-margin-stacked-separate vertical margin between the tabs and the preceeding content.
  • Use $sky-padding-even-large padding within the tab view, except when using a single full-width list within the tab.
